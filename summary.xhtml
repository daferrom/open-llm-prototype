Update to GitHub Workflow for Documentation Publishing

Summary

The changes refine the GitHub Actions workflow for documentation updates. The name of the workflow has been corrected, and a conditional step has been introduced to publish documentation when pull requests are successfully merged.

Changes

  • Modified Components: The file `.github/workflows/update_docs.yml` has been updated with the following changes:
    • The workflow name has been corrected from "CI Automation doc genetation" to "CI Automation doc generation".
    • A new trigger has been added to respond to closed pull requests.
    • The `publish` job has been renamed to `publish-doc-on-pr-merge`.
    • A condition has been added to check if a pull request was merged before executing the publish job.
  • Key Changes:
    • Added pull request integration for documentation publishing.
    • Improved naming clarity for the workflow and its jobs.
    • Added condition to publish documentation only when pull requests are merged, optimizing workflow triggers.

Impact

This update improves the CI/CD pipeline for documentation management. It enhances clarity in workflow and job naming and reduces unnecessary workflow runs by ensuring documentation is published only after pull requests are merged. Compatibility should not be affected significantly, and it brings better efficiency to the documentation update process.
